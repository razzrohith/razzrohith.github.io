QA checklist:

1. Public pages
Verify:
- Landing page loads.
- Browse Produce loads active Supabase listings.
- Produce Detail page loads.
- Public users can browse produce.
- No buyer phone numbers are publicly exposed.

2. Signup/login
Verify:
- Buyer signup works.
- Farmer signup works.
- Agent signup works.
- Admin signup is not available from frontend.
- Login works.
- Logout works.
- Header shows logged-in user name and role.
- No console errors.

3. Role route protection
Verify:
- Buyer cannot access Farmer Dashboard, Agent Dashboard, or Admin Dashboard.
- Farmer can access Farmer Dashboard.
- Farmer cannot access Agent Dashboard or Admin Dashboard.
- Agent can access Agent Dashboard.
- Agent cannot access Farmer Dashboard or Admin Dashboard.
- Admin can access Admin Dashboard.
- Admin can access Farmer Dashboard and Agent Dashboard only if current route protection intentionally allows admin override.

4. Role self-escalation
Try to update a non-admin user role to admin from the client/API.

Expected:
- Role update must fail.
- user_profiles.role must not change.
- Database should block role update with a column permission error or equivalent.

Also verify normal users can still update safe profile fields if any profile update helper exists:
- full_name
- phone
- village
- district
