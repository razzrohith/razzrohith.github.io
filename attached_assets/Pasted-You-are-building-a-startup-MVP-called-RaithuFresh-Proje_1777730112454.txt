You are building a startup MVP called RaithuFresh.

Project goal:
Build a responsive web app / PWA MVP for India, starting with Telangana. The app connects local fruit and vegetable farmers with nearby buyers. Farmers can sell a small part of their harvest directly to local customers before sending the rest to their agent, mandi, or wholesale buyer.

Important:
This MVP must focus ONLY on fruits and vegetables.
Do not add grains, dairy, meat, grocery, packaged food, delivery business, restaurant features, or ecommerce marketplace complexity.

Build type:
Create a clean, mobile-first responsive web app that can later become a mobile app.
Use simple UI, simple English, and a trustworthy farmer-friendly design.

Recommended stack:
- React
- TypeScript
- Vite
- Tailwind CSS
- Local mock data first
- No backend integration yet unless already configured
- Keep code modular and easy to connect to Supabase/Firebase later

Main user roles:
1. Buyer
2. Farmer
3. Agent
4. Admin

Core MVP pages/screens:

1. Landing Page
Purpose: collect interest and explain the idea.
Sections:
- Hero section with app name RaithuFresh
- Simple one-line pitch:
  "Buy fresh fruits and vegetables directly from nearby farmers."
- Problem section
- Solution section
- Benefits for farmers
- Benefits for buyers
- How it works
- Waitlist form with:
  - Name
  - Phone number
  - Role: Buyer / Farmer / Agent
  - Village/Town
- CTA buttons:
  - Join as Buyer
  - Join as Farmer

2. Buyer Home Page
Features:
- Show nearby produce listings
- Search by fruit or vegetable name
- Filter by:
  - Item type
  - Price
  - Distance
  - Harvest date
- Listing cards should show:
  - Produce name
  - Farmer name
  - Village/town
  - Price per kg
  - Quantity available
  - Harvest date/time
  - Distance
  - Button: Reserve
  - Button: Contact Farmer

3. Produce Detail Page
Show:
- Produce name
- Farmer name
- Farmer rating placeholder
- Available quantity
- Price per kg
- Harvest date/time
- Pickup location
- Basic quality notes
- Reserve quantity input
- Contact farmer button
- Pickup directions placeholder

4. Farmer Dashboard
Features:
- Farmer profile summary
- Add new fruit/vegetable listing form:
  - Produce name
  - Category: Fruit / Vegetable
  - Quantity available in kg
  - Price per kg
  - Harvest date/time
  - Pickup village/town
  - Contact phone
  - Quality notes
- Current listings table/cards
- Buttons:
  - Edit listing
  - Mark as sold
  - Mark out of stock
- Buyer interest section

5. Assisted Farmer Mode / Agent Dashboard
This is very important.
Some farmers may not use smartphones.
Build an Agent Dashboard where a local agent can update listings for assigned farmers.

Agent dashboard should include:
- Assigned farmer list
- Add listing for selected farmer
- Update stock for farmer
- Schedule callback request
- Call log notes
- Stock update history placeholder
- Commission tracking placeholder

6. Admin Dashboard
Features:
- Manage farmers
- Manage buyers
- Manage agents
- View all produce listings
- Verify farmer status
- Resolve issue/dispute placeholder
- Basic analytics cards:
  - Total buyers
  - Total farmers
  - Active listings
  - Reserved quantity
  - Estimated sales

7. Reservation Flow
Simple MVP only:
- Buyer clicks Reserve
- Show modal or page to enter:
  - Quantity needed
  - Buyer name
  - Buyer phone
- Confirmation message:
  "Your request has been sent to the farmer. Please contact the farmer before pickup."
- No online payment in MVP
- Payment method text:
  "Cash or UPI directly to farmer"

Design requirements:
- Mobile-first
- Clean cards
- Green/fresh agriculture theme
- Easy for normal Indian users
- Simple English
- Do not use complex corporate wording
- Use Telugu/Telangana-friendly feeling but keep app text in English for now
- Make it look like a real startup MVP, not a rough demo

Navigation:
Create a simple top navigation with:
- Home
- Browse Produce
- Farmer Dashboard
- Agent Dashboard
- Admin
- Join Waitlist

Mock data:
Create realistic Telangana sample data:
- Farmers from towns/villages around Telangana
- Fruits: mango, banana, guava, papaya, watermelon
- Vegetables: tomato, brinjal, green chilli, okra, onion, cucumber
- Prices in ₹ per kg
- Quantity in kg
- Distance in km
- Harvest date/time

Do not build:
- Full delivery system
- In-app payments
- Escrow
- Subscription plans
- AI prediction
- Complex logistics
- Grains/dairy/meat
- Real authentication in this first version
- Real database in this first version

Code requirements:
- Use reusable components
- Keep data in a mock data file
- Use clear folder structure
- Add comments only where needed
- Make sure the app runs without errors
- Make sure all buttons either work or show a clear placeholder message
- Make sure forms validate required fields
- Make sure the design works on mobile and desktop

Expected final output:
- A working responsive RaithuFresh MVP web app
- Clean homepage
- Buyer browse flow
- Farmer listing dashboard
- Agent-assisted farmer dashboard
- Admin dashboard
- Mock reservation flow
- No broken pages
- No console errors

After building, report:
- Files created/changed
- How to run the app
- Main features completed
- Any missing items
- Any errors or limitations
