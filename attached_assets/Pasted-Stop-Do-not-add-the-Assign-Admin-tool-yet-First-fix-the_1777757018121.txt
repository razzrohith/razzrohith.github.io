Step 2 — Create security patch

Create:

supabase/patch-user-role-security.sql

Patch requirements:

1. Normal authenticated users can update only safe profile fields:
- full_name
- phone
- village
- district

2. Normal authenticated users must NOT be able to update:
- id
- role
- created_at

3. Keep users able to read their own profile.

4. Keep users able to insert their own profile during signup.

5. Do not allow public/anon users to update user_profiles.

6. Do not weaken admin route protection.

7. Do not expose service role key.

8. Do not use SUPABASE_DB_URL in frontend.

Preferred database-level fix:
Use column-level GRANTs where possible:

- REVOKE broad UPDATE on user_profiles from authenticated
- GRANT UPDATE(full_name, phone, village, district) ON user_profiles TO authenticated

Then keep/update the RLS policy so users can update only their own row:

auth.uid() = id

If column-level privileges are not enough due to existing grants, remove/recreate the broad grant safely.
