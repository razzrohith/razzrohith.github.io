QA checklist:

1. Public pages

- Buyer phone numbers are not publicly exposed.

2. Signup/login
Verify:
- Buyer signup works.
- Farmer signup works.
- Agent signup works.
- Admin signup is not available from frontend.
- Login works.
- Logout works.
- Header shows logged-in user name and role.
- No console errors.

3. Route protection
Verify:
- Buyer cannot access Farmer Dashboard.
- Buyer cannot access Agent Dashboard.
- Buyer cannot access Admin Dashboard.
- Farmer can access Farmer Dashboard.
- Farmer cannot access Agent Dashboard.
- Farmer cannot access Admin Dashboard.
- Agent can access Agent Dashboard.
- Agent cannot access Farmer Dashboard.
- Agent cannot access Admin Dashboard.
- Admin can access Admin Dashboard.
- Admin can access Farmer Dashboard if admin override is intentionally allowed.
- Admin can access Agent Dashboard if admin override is intentionally allowed.

4. Role self-escalation security
Try to update a non-admin user role to admin from the client/API.

Expected:
- Role update must fail.
- user_profiles.role must not change.
- Database should block role update with a column permission error or equivalent.

Also verify:
- Authenticated users cannot update role.
- Authenticated users cannot update created_at.
- Authenticated users cannot update id.
- Authenticated users can still update safe profile fields if a helper/form exists:
  - full_name
  - phone
  - village
  - district
