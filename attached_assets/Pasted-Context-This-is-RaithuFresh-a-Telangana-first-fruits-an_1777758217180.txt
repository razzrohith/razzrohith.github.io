Step 1 — Create SQL patch

Create:

supabase/patch-reservation-update-security.sql

Required changes:

A. reservations column-level update restriction

Safely revoke broad UPDATE on reservations:

- REVOKE UPDATE ON reservations FROM anon
- REVOKE UPDATE ON reservations FROM authenticated

Then grant only:

- GRANT UPDATE(status) ON reservations TO authenticated

Do not allow UPDATE on:
- id
- listing_id
- buyer_name
- buyer_phone
- quantity_kg
- payment_method
- created_at

Keep existing RLS policies that allow:
- farmer_update_own_reservation_status
- admin_update_reservation_status

Recreate them only if needed, and keep them scoped:
- Farmer can update reservation status only for reservations connected to their own listings.
- Admin can update reservation status only when user_profiles.role = 'admin'.
- Status must stay within:
  - pending
  - confirmed
  - cancelled
  - completed

B. produce_listings leftover cleanup

If authenticated has DELETE on produce_listings, revoke it:

- REVOKE DELETE ON produce_listings FROM authenticated

Do not add a DELETE policy.
Do not break Farmer Dashboard.
