On the Landing Page waitlist form, the user selects a role from the “I am a” dropdown, but the form still shows two submit buttons:

UX audit checklist:
Look for issues like:
- Buttons that conflict with selected dropdown values.
- Duplicate CTAs that confuse the user.
- Buttons that appear for the wrong role.
- Submit buttons that say one thing but save another value.
- Labels that do not match actual behavior.
- Actions that are visible but cannot work.
- Wrong routes or broken links.
- Public pages exposing private-like information.
- Logged-out users seeing dashboard-like actions without explanation.
- Buyer/farmer/admin wording mixed incorrectly.
- “Payment” wording that sounds like online payment when MVP only supports Cash/UPI direct to farmer.
- “Delivery” wording anywhere, since MVP is pickup only.
- Map wording that implies embedded maps or paid map API.
- Contact wording that exposes buyer phone publicly.
- Admin-only wording visible to normal users.
- Agent actions visible to buyer/farmer users.
- Confusing empty states.
- Success messages that do not explain next step.
- Form validation messages missing or unclear.
- Buttons that should be disabled based on status but are still active.
- Cards showing “Reserve” when item is sold/out_of_stock.
- Any old mock/demo wording that should not be visible in live Supabase mode.
- Any “DealNest”, “deal”, “coupon”, “store”, or unrelated marketplace wording.
